# Agent Learning (智能体交互可视化解析工具)

本项目是一个专为智能体（Agent）开发学习者、职业教育及课堂教学设计的可视化网络抓包与分析工具。

在智能体开发教学中，Agent 与大模型之间的底层通信（如 Prompt 组装、工具调用、Token 消耗）通常是一个“黑盒”。本项目通过轻量级的代理技术，**无侵入地捕获并解析 API 流量**，将抽象的数据流具象化，并借助 AI 进行智能分析，显著降低初学者的认知门槛。

## ✨ 核心特性

* **🔍 无侵入式流量捕获**：基于 `mitmproxy` 自动拦截本地 Agent 与云端大模型的通信流量（支持 OpenAI、Anthropic 格式及各类兼容 API）。
* **🌳 数据报文可视化**：将复杂晦涩的 JSON 格式请求与响应数据，转换为结构清晰的树状图，轻松查看 `function_call`、`reasoning` 等关键字段。
* **🤖 AI 智能导师分析**：内置 AI 解析模块，可接入各大模型 API（如阿里云、OpenAI 等），以“导师”视角对抓取到的交互日志进行通俗易懂的原理剖析。
* **⚙️ 灵活的模型管理**：支持自定义配置多个大模型接口地址与 API Key，一键测试连接状态。
* **🛡️ 完整会话记录**：自动保存历史会话，支持离线回顾、多角度对比分析，便于生成教学案例。

## 🛠️ 应用场景

* **教育与培训**：作为人工智能基础课程或职业技能培训的辅助工具，直观演示智能体的思考逻辑与决策过程。
* **本地 Agent 调试**：帮助开发者定位提示词组装错误、工具调用异常等底层问题。
* **安全与合规审计**：监控本地应用向云端发送的数据内容。

## 🚀 快速开始

### 1. 环境准备

确保你的系统已安装 Python 3.10 或以上版本。

克隆本仓库到本地，并安装依赖：

```bash
git clone https://github.com/sdnjyjm/agent_learning.git
cd llm-traffic-inspector
pip install -r requirements.txt

```

> **注：** 项目当前 GUI 基于 `PyQt5` 构建，若你后续计划重构以避免特定的 DLL 缺失错误，可按需平滑迁移至 `PySide6` 并更新依赖清单。

### 2. 证书配置 (重要)

由于大部分大模型 API 使用 HTTPS 加密，你需要安装 `mitmproxy` 的根证书才能解密流量。

1. 首次运行项目后，`mitmproxy` 会在用户目录下（如 `~/.mitmproxy`）生成证书。
2. 在软件主界面点击 **“导出MITM证书”**。
3. 双击导出的证书文件，将其安装到操作系统的 **“受信任的根证书颁发机构”** 中。

### 3. 运行程序

运行主程序（无需管理员）：

```bash
python main.py

```

### 4. 使用流程

1. **设置系统代理**：在软件主界面点击“设置系统代理”（软件将在 `127.0.0.1:8080` 开启监听）。
2. **配置 AI 解析模型**：点击“大模型管理”，添加你用于解析日志的大模型 API（如阿里云 DashScope 或其他兼容接口）。
3. **运行你的 Agent**：在本地运行你的智能体代码（如 Qwen-Agent、LangChain 等脚本）。
4. **查看与分析**：回到本软件，你将看到实时捕获的交互记录。右键点击特定记录，选择 **“AI解析会话内容”**，让大模型为你解释当前交互的原理。

## 📁 核心文件结构

* `main.py` / `main_window.py`：程序入口与 PyQt5 主界面。
* `proxy_handler.py`：系统代理设置与 `mitmdump` 子进程管理。
* `mitm_addon.py`：核心抓包逻辑，负责流式解析（Streaming）和非流式 API 报文的拦截与格式化。
* `ai_analyzer.py`：AI 智能解析模块，构建 Prompt 并调用大模型分析本地交互日志。
* `llm_manager.py`：大模型配置管理与连通性测试。
* `session_flow_window.py`：双栏树状 JSON 报文查看器。
